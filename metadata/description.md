# *, Actually — description

A domain-blind engine that weaves a directory of depth-layered markdown into a fully static, navigable, terminal-aesthetic knowledge site.

Most technical documentation is written by the people who built the thing for the people who already understand the field, and it fails the actual job: getting someone from "I have a question" to "I understand enough to act." *, Actually is the answer, and star-actually is its engine — a build system that renders knowledge organized by question and relationship rather than by product surface, giving every concept depth layers (name, definition, usage, relationships, full theory) the reader can dial through at will. The engine is domain-blind: it never mentions any subject, and everything specific arrives from an instance repo through two files — a site.yaml and a directory of depth-layered markdown nodes. It has three parts: the Loom parses and validates the node graph and renders the static site; the Terminal is the reading experience — HTML, CSS, one vanilla JavaScript file, and vendored htmx, with zero external requests; and the Receptionist is an optional FastAPI service that maps a natural-language question to an entry node, with the site degrading gracefully to a browsable graph and full-text search when it is absent. The deeper argument is about reader agency: the reader of a typical man page has none, and *, Actually is what reader agency looks like when the architecture is built to support it. SSH, Actually and Ho System, Actually — the latter a guide to the Ho System itself — are its live instances, served as static sites at star-actually.sageframe.net; each subject is a separate content repo that pins and consumes this shared engine.

Python 3.12+ (the Loom and CLI on pyyaml, markdown-it-py, and jinja2) that outputs a static, htmx-based site, with an optional FastAPI + Anthropic "Receptionist" service; MIT-licensed, published as the star-actually package and consumed by per-subject instance repos.
